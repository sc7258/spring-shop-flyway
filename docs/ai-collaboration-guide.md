# AI Collaboration Guide (Role-Based)

이 문서는 AI 어시스턴트(`@04-roles.md`)에게 적절한 역할을 부여하여, 프로젝트의 다양한 관점에서 전문적인 도움을 받는 방법을 설명합니다.

## 1. 역할 기반 협업의 필요성
AI는 기본적으로 "도움이 되는 어시스턴트" 모드로 동작하지만, 명확한 페르소나(Persona)를 부여하면 답변의 품질과 깊이가 달라집니다.
- **일반 요청:** "로그인 기능 만들어줘." -> 기능 구현에만 집중.
- **역할 지정:** "**@보안담당자** 입장에서 로그인 기능을 만들어줘." -> SQL Injection, XSS, 비밀번호 암호화 등을 고려한 안전한 코드 제공.

## 2. 주요 역할 및 활용 예시

### 2.1 Product Manager (PM)
- **상황:** 기획 단계, 요구사항 정의, 우선순위 결정.
- **명령어 예시:**
  - "@04-roles.md **PM**으로서 이번 스프린트의 목표를 설정해줘."
  - "사용자 입장에서 이 기능이 정말 필요한지 **기획자** 관점에서 비판해줘."

### 2.2 Software Architect (아키텍트)
- **상황:** 시스템 설계, DB 모델링, 기술 스택 선정.
- **명령어 예시:**
  - "@04-roles.md **아키텍트**로서 이 DB 스키마의 확장성을 검토해줘."
  - "MSA로 전환할 때 발생할 수 있는 문제를 **아키텍트** 관점에서 분석해줘."

### 2.3 Backend Developer (개발자)
- **상황:** 코드 구현, 버그 수정, 리팩토링.
- **명령어 예시:**
  - "**백엔드 개발자**로서 이 코드를 클린 코드 원칙에 맞춰 리팩토링해줘."
  - "TDD 방식으로 이 기능을 구현해줘."

### 2.4 Code Reviewer (리뷰어)
- **상황:** 코드 품질 검증, 보안 점검.
- **명령어 예시:**
  - "@04-roles.md **리뷰어** 입장에서 내 코드를 아주 깐깐하게 비판해줘."
  - "보안 취약점이 없는지 **보안 담당자** 관점에서 체크해줘."

### 2.5 Technical Writer (테크니컬 라이터)
- **상황:** 문서 작성, 용어 정리, 릴리즈 노트.
- **명령어 예시:**
  - "**테크니컬 라이터**로서 이 API 문서를 초보자도 이해하기 쉽게 다듬어줘."
  - "릴리즈 노트를 작성해줘."

## 3. 팁 (Tips)
- **역할 전환:** 하나의 대화 세션 안에서도 역할을 바꿔가며 요청할 수 있습니다.
  - "먼저 **아키텍트**로서 설계를 해주고, 그 다음 **개발자**로서 코드를 짜줘."
- **복합 역할:** 두 가지 관점을 동시에 요구할 수도 있습니다.
  - "**PM**과 **개발자**가 논의하는 형식으로 이 기능의 장단점을 분석해줘."
